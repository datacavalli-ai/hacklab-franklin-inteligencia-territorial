Módulos y aplicación al MVP

Clase 01

Fundamentos de IA y Estrategias de Prompting

¿Qué trata?

Uso estratégico de IA generativa, plataformas clave e ingeniería de prompts para optimizar análisis, documentación y toma de decisiones.

Aplicación Franklin

Crear prompts reutilizables para resumir terreno, generar reportes, interpretar indicadores, preparar minutas y apoyar la comunicación con municipio y HUB STGO.

Preparación

  • Definir 5 prompts base del proyecto.
  • Documentar hipótesis territorial actual.
  • Preparar ejemplos de datos de terreno.
IA aplicadaProductividadReportes
Clase 02

Gestión de Datasets y Generación de Código en Lenguaje Natural

¿Qué trata?

Estructuras de datos, modelos aplicados y generación automatizada de código mediante lenguaje natural.

Aplicación Franklin

Ordenar datasets de visitas, puntos críticos, evidencia fotográfica, capas municipales y datos abiertos para automatizar análisis en Python/Colab.

Preparación

  • Consolidar diccionario de datos del MVP.
  • Separar datos de terreno, externos y municipales.
  • Identificar campos mínimos para modelar criticidad.
DatasetsPythonAutomatización
Clase 03

Análisis Urbano con Python y Operaciones Espaciales

¿Qué trata?

Ecosistema Python aplicado al urbanismo, operaciones geométricas, sistemas de coordenadas e integración de capas geoespaciales.

Aplicación Franklin

Fortalecer el núcleo técnico del MVP: AOI Franklin, capas OSM, puntos de microbasurales, heatmaps, distancia a equipamientos y análisis por manzana.

Preparación

  • Revisar GeoPandas, Shapely, Folium y CRS.
  • Validar coordenadas WGS84 y capas base.
  • Preparar preguntas sobre buffers, intersecciones y densidades.
GeoPandasCRSHeatmap

Inteligencia Colectiva y Relevamiento de Datos Geoespaciales

¿Qué trata?

Crowdsourcing, mapeo participativo y herramientas digitales para recolección comunitaria de información con rigor geoespacial.

Aplicación Franklin

Diseñar un mecanismo de reporte ciudadano georreferenciado para complementar observaciones de terreno y registros institucionales.

Preparación

  • Definir qué datos podría aportar la comunidad.
  • Diseñar formulario mínimo de reporte.
  • Revisar criterios de validación de evidencia.
Ciencia ciudadanaCrowdsourcingTerreno
Clase 05

Infraestructura IoT y Procesamiento de Lenguaje para la Participación

¿Qué trata?

Integración de sensores ciudadanos, dispositivos IoT y Procesamiento de Lenguaje Natural para sistematizar participación ciudadana.

Aplicación Franklin

Explorar una fase futura con sensores, reportes ciudadanos, cámaras o texto de denuncias para alimentar alertas tempranas y priorización municipal.

Preparación

  • Identificar variables capturables por sensores.
  • Listar fuentes textuales: denuncias, formularios, redes, app municipal.
  • Separar MVP actual de fase futura IoT/PNL.

Clase 06

Análisis de Sentimiento e Integración Small & Big Data

¿Qué trata?

Análisis cualitativo en redes sociales y consultas públicas, integrando datos comunitarios con grandes volúmenes institucionales.

Aplicación Franklin

Combinar evidencia de terreno, percepción ciudadana, datos municipales y datos abiertos para enriquecer el índice de criticidad territorial.

Preparación

  • Definir Small Data: observaciones, fotos, comunidad.
  • Definir Big Data: registros municipales, censo, OSM, denuncias.

Clase 08

Implementación Estratégica: Del Piloto a la Escala

¿Qué trata?

Hoja de ruta para adopción de IA organizacional, diseño, ejecución y evaluación de pilotos, presentación final y defensa de resultados.

Aplicación Franklin

Convertir el piloto de Barrio Franklin en una metodología replicable para otras zonas, comunas o problemáticas ambientales urbanas.

Preparación

  • Definir estado actual TRL/MVP.
  • Construir hoja de ruta 30-60-90 días.
  • Preparar propuesta de escalamiento y métricas de impacto.
EscalamientoTRLImpacto

Relación con el modelo de decisión territorial

El objetivo no es sólo mapear microbasurales, sino construir un sistema de priorización territorial basado en evidencia. Cada módulo fortalece una dimensión del modelo:

Índice de Criticidad Territorial = f( evidencia de terreno, intensidad espacial, recurrencia, volumen estimado, obstrucción peatonal, proximidad a equipamientos, percepción ciudadana, registros institucionales, capacidad de respuesta municipal )

Preguntas guía para mentores

  • ¿Qué variables mínimas hacen robusto el índice de criticidad?
  • ¿Cómo validar datos de ciencia ciudadana sin perder rigor geoespacial?
  • ¿Qué nivel de predicción es razonable para un MVP temprano?
  • ¿Cómo pasar de Franklin a una metodología replicable?
